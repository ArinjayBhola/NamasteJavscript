Async/Await waits at the point of time. The program will wait

When the async function is called the function gets pushed to call stack and as soon as the js engine seens await in the function the function gets suspended until the promise in the function gets resolved after that it will execute the code of next line of function. After some time or promise gets resolved the function again comes in call stack and will start executing where it left. If there is another promise then it will again suspend the function once the promise gets resolved then the function will again come to call stack and execute where it left.It happens so that the main thread do not gets blocked

Actually call stack is not only that suspends the function => The function suspends itself when it encounters an await, but this behavior is coordinated by the JavaScript engine (including the call stack and event loop).

If there is another promise in same function then it will also starts resolving as soon as function gets invoked. When js engine reaches that line it sees that if the promise is resolved then it prints it.

Async/Await is syntactical sugar over .then().catch()
